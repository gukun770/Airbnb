What are the features in my model?¶

Since I want to train a model for new host and predict his/her future revenue if he/she choose to use Airbnb service. Given this purpose, a lot of columns are not fit for first time users, so any review-related columns are excluded in my model, even though they contains very strong signals.

After multiple trials, I selected the following columns to train my model.

Features: 'price','room_type','guests_included','bedrooms','cleaning_fee', 'latitude','longitude','zipcode',

Target: available_30

  • I train the model using all the data before 2017-11-01 and test the model using data after this day.
  • Target is the total number of available days for next 30 days given that scraped days.
  • Data is scraped from airbnb website at the beginning of every month from 2017 Jan to 2018 May.

X_train, X_test, y_train, y_test = F.my_train_test_split(listings, '2017-11-01')
p = Pipeline([
    ('pre_select', PreselectColumns()),  # Select columns to use in the training set
    ('FeatureEnginner', FeatureEnginner()), # Calculate price per person, perice per bedroom, distance from center, etc. 
    ('PricePerBedroom', PricePerBedroom()), # calculate difference price per night vs average neighbourhood (zipcode)
    ('DropColumns', DropColumns()), # drop zipcode
    ('Getdummies', Getdummies()), # dummify categorical variable, drop not contributing columns 
])
mod = p.fit(X_train,y_train)
X_train_post = mod.transform(X_train)
X_test_post = mod.transform(X_test)


rf = RandomForestRegressor(n_estimators=500)
rf.fit(X_train_post,y_train)


y_pred = rf.predict(X_test_post)

# Plot permutational feature importance using mean squared log error as metrics
neg_scorer = lambda y1, y2: mean_squared_log_error(y1, y2)*(-1)
feature_importance = F.permutation_importance(rf, X_test_post.values, y_test, scorer=neg_scorer)
series_feature_importance = pd.Series(feature_importance,index=X_train_post.columns.tolist())
feature_importance = series_feature_importance.copy()

F.plot_feature_importance(feature_importance)
  • guests_included: number of guests
  • room_type_Entire_home/apt: whether this listing is about entire home/apt or not.
  • diff_price_per_bedroom_hood: Difference between the listing's price per bedroom compared to the average price per bedroom of the same neighborhood.
  • latitude: latitude of that listing
  • price_per_bedroom: price divided by the number of bedrooms
  • distance: distance from the listings to the center of San Francisco city.
  • longitude: longitude of that listing
  • cleaning_fee_person: cleaning feeing divided by number of guests (guest included)
  • cleaning_fee: flat cleaning fee charged at checkout.
  • price: price per night.

Since the number of bedrooms feature appears both in the difference of price per bedroom compared to its neighborhood average level and price per bedroom, bedrooms itself becomes no longer important.

p = Pipeline([
    ('pre_select', PreselectColumns()),  # Select columns to use in the training set
    ('FeatureEnginner', FeatureEnginner()), # Calculate price per person, perice per bedroom, distance from center, etc. 
    ('PricePerBedroom', PricePerBedroom()), # calculate difference price per night vs average neighbourhood (zipcode)
    ('DropColumns', DropColumns()), # drop zipcode
    ('Getdummies', Getdummies()), # dummify categorical variable, drop not contributing columns 
    ('Interaction', Interaction()), # Feature engineered some interaction between features
])
mod = p.fit(X_train,y_train)
X_train_post_lr = mod.transform(X_train)
X_test_post_lr = mod.transform(X_test)


lr = Ridge(alpha=0.0122,normalize=True)
lr.fit(X_train_post_lr,y_train)


y_pred = lr.predict(X_test_post_lr)


neg_scorer = lambda y1, y2: mean_squared_log_error(y1, np.clip(y2, 0, 30))*(-1)
feature_importance = F.permutation_importance(lr, X_test_post_lr.values, y_test, scorer=neg_scorer)
series_feature_importance = pd.Series(feature_importance,index=X_train_post_lr.columns.tolist())
feature_importance = series_feature_importance.copy()

F.plot_feature_importance(feature_importance)

Since Ridge is a linear model, I feature engineered some interaction between boolean features and numeric features.

  • price_entire_home: price * room_type_entire_home
  • cleanfee_entire_home: cleanning_fee * room_type_entire_home
